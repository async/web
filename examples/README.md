# Examples

These examples document the public Async Web shapes. The browser shell fixtures that exercise the full route graph still live in the workspace-local WebRuntime package until they are promoted into standalone example packages.

| Example | Focus |
| --- | --- |
| [hello-app](hello-app/README.md) | minimal `@async/web` app defaults |
| [simple-fullstack](simple-fullstack/README.md) | frontend plus `/api` runtime routing |
| [service-worker-cache](service-worker-cache/README.md) | service-worker-style cache placement |
| [cdn-edge-cache](cdn-edge-cache/README.md) | shared edge cache policy |
| [edge-middleware](edge-middleware/README.md) | redirect, rewrite, and middleware flow |
| [multi-app-network](multi-app-network/README.md) | multiple apps in one route graph |
| [async-db-app](async-db-app/README.md) | AsyncDB route adapter placement |
| [runtime-routing](runtime-routing/README.md) | explicit WebRuntime route control |
